# 東京バドミントン体育館検索

東京都内でバドミントンの個人利用が可能な体育館・スポーツ施設を検索できるWebアプリケーションです。

---

## できること

- **施設の一覧表示・検索**
  - キーワード（施設名・住所・最寄り駅）での絞り込み
  - 区での絞り込み（中央区・文京区・台東区など19区対応）
  - 公式サイトあり/なしでの絞り込み

- **施設の詳細確認**
  - 施設名・住所・郵便番号・最寄り駅・電話番号を表示
  - 公式サイトへのリンク
  - 区の施設予約ポータルへのリンク

- **管理画面**
  - 施設データのCRUD（登録・編集・削除）
  - 施設情報の手動管理

- **データの自動インポート**
  - 東京都オープンデータ（19区分）からバドミントン利用可能施設を一括取り込み
  - HeartRails APIで最寄り駅を自動補完（緯度経度または郵便番号から取得）
  - 区の予約ポータルURLを一括設定

---

## できないこと

- **曜日・時間帯での絞り込み**
  - 東京都オープンデータにはバドミントン個人利用の開放スケジュールが含まれないため未実装

- **リアルタイムの空き状況確認**
  - 各区の予約ポータルへのリンクを提供するが、空き状況の取得には対応していない

- **全23区の網羅**
  - 現在は東京都オープンデータが提供されている19区のみ対応（千代田区・新宿区・北区・港区・江戸川区など一部の区はデータ未提供）

- **施設ごとの予約可否の正確な判断**
  - 区の予約ポータルは区単位の紐付けのため、指定管理者が運営する施設は別の予約サービスを使っている場合がある

## 将来拡張できること

- **曜日・時間帯フィルター** — 施設ごとにスケジュールデータを手動登録することで実現可能（`GymSchedule`テーブルは実装済み）

- **対応区の拡大** — データが提供されていない区の施設を手動登録
- **PWA対応** — オフライン表示・ホーム画面追加（マニフェストファイルは既存）

## ローカル起動方法

### 前提条件

- Docker / Docker Compose がインストール済みであること

### 手順

```bash
# 1. リポジトリをクローン
git clone <リポジトリURL>
cd search-gym-in-tokyo

# 2. コンテナを起動（初回はイメージビルドあり）
docker-compose up

# 3. 別ターミナルでDBのセットアップ
docker-compose exec web rails db:create db:migrate

# 4. 東京都オープンデータから施設データをインポート
docker-compose exec web rails gym:import_from_open_data

# 5. 区の予約ポータルURLを一括設定
docker-compose exec web rails gym:update_reservation_urls
```

ブラウザで http://localhost:3000 にアクセスすると施設一覧が表示されます。

管理画面は http://localhost:3000/admin/gyms です。

### その他のRakeタスク

```bash
# 最寄り駅が未登録の施設を補完（HeartRails API使用）
docker-compose exec web rails gym:enrich_stations
```

## データソース・ライセンス

- 施設データ：[東京都オープンデータカタログ](https://catalog.data.metro.tokyo.lg.jp/) 掲載の各区スポーツ施設CSVを使用
- 最寄り駅情報：[HeartRails Express](https://express.heartrails.com/) を使用
- 出典：「位置参照情報ダウンロードサービス」（国土交通省）を加工して作成
